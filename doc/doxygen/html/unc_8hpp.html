◆ unc_types

enum unc_types

16 type bits used by unc uncTypes. Bit set = 1 means a positive attribute.

Enumerator
VALUE_ZERO 

bit zero = 1 means value is zero.

VALUE_INTEGER 

Exact integer 2,3,4 ... may not be possible to store `int` into `double`?

(Too near zero to allow relsd = sd /mean.) A few times std::numeric_limits<float>::min() may be a suitable limit?

VALUE_RATIONAL 

Integers & Fractions 1/2, 1/3, 2/3, 5/4 ... so don't need irrational - must be if not rational. Irrational like pi, e ... represented as accurately as possible by the double floating-point type, usually for 64-bit double, 15 to 17 decimal digits.

VALUE_NEGATIVE_ONLY 

Value can ONLY be < 0.. ? or <=0?

VALUE_POSITIVE_ONLY 

Value can ONLY be >=0..

UNC_KNOWN 

Uncertainty known (but if not, value may still be known OK).

UNC_NOPLUS 

Uncertainty can only be < value, + = zero.

UNC_NOMINUS 

Uncertainty can only be > value - = zero. Integer, Rational or Exact by definition like 25.4 mm per inch, if unc = 0.0f or both UNC_NOPLUS & UNC_NOMINUS.

UNC_QUAN_DECIMAL 

Quantised by least significant decimal digit.

UNC_QUAN_BINARY 

Quantised by least significant binary digit.

UNC_EXPLICIT 

If bit set = 1, uncertainty implicit from read significant digits, for example: unc_input of 10.0 will set this bit and set uncertainty = 0.05, or explicit if +/- or calculated (normal default case, bit = 0).

UNC_UNIFORM 

Uncertainty has rectangular or uniform probability distribution.

UNC_TRIANGULAR 

Uncertainty has triangular probability distribution.

DEG_FREE_EXACT 

If not rectangular or triangular bits set, then normal or Gaussian distribution, the default. Both bits 11 and 12 == 1 then has distribution un-defined as yet. Distribution type is used to estimate confidence intervals and rounding. Known number of observations -1,

DEG_FREE_KNOWN 

Degrees of freedom is defined.

so default zero means 1 observation. else estimated, for example, using Welch-Satterthwaite.

SPARE 

This last unc_type is spare.

enum uncertainflags

Control of printing uncertain values, similar to ios flags.

Enumerator
defaults 

Default.

firm 

bit 0: == 0 == false means flex, or firm == 1 == true.

setScaled 

bit 1 Set scaled == 1 or not scaled == 0.

autoScaled 

bit 2: auto = 1 with << autoscale ...

by fixed factor SetScale and << scale ... power10 in range -max to +max.

plusMinus 

bit 3 = 1 means add +/- uncertainty Usage:

std::cout << plusminus << u;
addSISymbol 

bit 4 = 1, add suffix SI symbol G, M, k, m, u, n, p ...

addSIPrefix 

If one is applicable, else = 0 do nothing.

bit 5 = 1, add suffix SI prefix Giga, Mega, kilo ...

noisyDigit 

Add extra 'noisy' decimal digit to value & sd. Usage: std::cout `<< addnoisyDigit << u;` This means is suitable for input to other calculations because the random extra 2 to 3 bits will approximate a continuous function. Quantization to fewer digits would distort statistical calculations. Default is to use minimum decimal digits for clearer display. Extra digit also added when degrees of freedom > 10.

useSetSigDigits 

if bit 7 = 1 use set sig digits else calculate from sd(isUseSetSigDigits).

useSetUncSigDigits 

if bit 8 = 1 use set Unc sigfig (isStdDevSigDigitsSet).

degfree 

else calculate from degrees of freedom.

0x200, If bit 9 == 1, add output of degrees of freedom as (99).

replicates 

0x400 If == 1, add output of replicates as [99] if > 1.

limits 

0X800 add limits.

Uncertainties ARE correlated. This is the unusual case where the sum must be exact, so the uncertainties are subtracted. Example: two blocks which fit into a box perfectly. So if both have an uncertainties, they must cancel when the uncertainties are added. Also applies to items like concentrations which must add up to 100% or unity.
